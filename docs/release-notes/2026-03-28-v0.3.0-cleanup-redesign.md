# v0.3.0 - Cleanup Redesign

本版本把 CodexSubMcp 从“只处理 orphan suite 的 GUI 工具”升级为“围绕当前态观测、预览式清理、历史审计”的单一主流程工具。

## 主要更新

- 新增统一主流程：`刷新 → 预览 → 清理 → 日志`
- MCP 配置读取改为 Codex 官方来源：
  - `~/.codex/config.toml`
  - 项目 `.codex/config.toml`
- 新增 `state_5.sqlite` open 子代理统计
- 新增当前运行态分析：
  - 运行中子代理数
  - 运行中 suite 数
  - 运行中 MCP 实例数
  - 已配置 MCP 数
  - drift
- 新增 `stale_attached_branch` 清理目标
  - 在同一 live Codex 会话内按 `tool_signature` 分组
  - 始终保留最新 branch
  - 只清理更老的 stale branches
- 新增三类结构化日志：
  - `refresh`
  - `preview`
  - `cleanup`
- 新增累计清理统计：
  - cleanup 次数
  - closed suite
  - closed stale branch
  - killed MCP instance
  - killed process
- GUI 页面职责重构：
  - 总览：当前态 + 最近结果 + 累计统计
  - MCP：configured / running / drift
  - 清理：preview targets
  - 日志：refresh / preview / cleanup 历史回放

## 兼容与变更说明

- `scan mcp --format json` 不再输出 `installed_candidates`
- `dry-run` 现在作为 `preview` 的兼容入口
- GUI 中不再保留独立“扫描 MCP”工作流
