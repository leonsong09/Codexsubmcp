# CodexSubMcp v0.3.0 已发布

如果你在 Windows 上长期用 Codex 跑多 agent / 子代理，遇到 `node.exe`、MCP 进程或旧 branch 残留越来越多，这一版就是为这个问题准备的。

## 这次版本能帮你什么

- 把操作流程收口为更直接的主线：`刷新 → 预览 → 清理 → 日志`
- 直接展示当前最关心的运行态：
  - 正在运行的子代理数量
  - 正在运行的 suite 数量
  - 正在运行的 MCP 实例数量
  - 已配置 MCP 数量
- 支持两类清理目标：
  - `orphan_suite`
  - `stale_attached_branch`
- MCP 页面只保留真正有判断价值的信息：
  - configured
  - running
  - drift
- 总览页新增累计清理统计，更容易看清这段时间到底清掉了多少内容
- 仍然保持单文件 `CodexSubMcpManager.exe`，没有 Python 环境也可以直接运行

## 适合谁升级

- Windows 上使用 Codex 并行开发 / 多 agent 工作流的人
- 经常看到残留 `node.exe` / MCP 进程持续占内存的人
- 需要先看当前态，再决定是否清理的人
- 想把清理动作沉淀为更稳定 GUI 流程的人

## 怎么开始用

1. 打开 Release 页面下载 `CodexSubMcpManager.exe`
2. 双击启动 GUI
3. 在 `总览` 页先执行一次 `刷新`
4. 到 `清理` 页执行 `预览清理`
5. 确认目标后再执行正式清理
6. 需要长期后台巡检时，可在 `计划任务` 页安装 `CodexSubMcpWatchdog`

## 升级后你会看到的变化

- “扫描 MCP”不再作为单独工作流保留
- `dry-run` 现在作为 `preview` 的兼容入口
- GUI 会更强调“当前正在发生什么”和“最近清理了什么”
